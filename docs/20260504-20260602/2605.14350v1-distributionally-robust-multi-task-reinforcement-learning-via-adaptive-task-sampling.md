## 摘要
多任务强化学习旨在训练单个智能体同时高效优化多个任务的性能。然而，联合优化所有任务通常会导致学习不平衡：智能体快速解决简单任务，但在困难任务上学习缓慢。虽然先前工作主要将此不平衡归因于冲突的任务梯度，并提出梯度操纵或专门架构来解决，但我们反而关注一个不同且尚未充分探索的挑战：不平衡的数据分配。标准的多任务强化学习为每个任务分配相等数量的环境交互，这导致过度分配数据给只需少量交互即可解决的简单任务，而分配不足给需要更多经验才能解决的困难任务。为应对这一挑战，我们引入了分布鲁棒自适应任务采样（DRATS），一种自适应地优先采样离解决最远的任务的算法。我们通过将多任务强化学习形式化为一个可行性问题来推导DRATS，从中我们推导出一个极小极大目标，用于最小化最差情况下的回报差距，即期望目标回报与智能体在任务上获得的回报之间的差异。在MetaWorld-MT10和MT50等基准测试中，与现有任务采样算法相比，DRATS提高了数据效率并增加了最差任务的性能。

## Abstract
Multi-task reinforcement learning (MTRL) aims to train a single agent to efficiently optimize performance across multiple tasks simultaneously. However, jointly optimizing all tasks often yields imbalanced learning: agents quickly solve easy tasks but learn slowly on harder ones. While prior work primarily attributes this imbalance to conflicting task gradients and proposes gradient manipulation or specialized architectures to address it, we instead focus on a distinct and under-explored challenge: imbalanced data allocation. Standard MTRL allocates an equal number of environment interactions to each task, which over-allocates data to easy tasks that require relatively few interactions to solve and under-allocates data to hard tasks that require substantially more experience to solve. To address this challenge, we introduce Distributionally Robust Adaptive Task Sampling (DRATS), an algorithm that adaptively prioritizes sampling tasks furthest from being solved. We derive DRATS by formalizing MTRL as a feasibility problem from which we derive a minimax objective for minimizing the worst-case return gap, the difference between a desired target return and the agent's return on a task. In benchmarks like MetaWorld-MT10 and MT50, DRATS improves data efficiency and increases worst-task performance compared to existing task sampling algorithms.

- **整体含义**：本文提出通过自适应任务采样来优先与最困难（离解决方案最远）的任务交互，从而更高效地利用数据，提升所有任务尤其是最差任务的性能，最终实现更平衡的多任务学习。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将多任务强化学习形式化为一个**可行性问题**：寻找参数 θ 使得在所有任务上的回报都达到或超过一个目标回报 \(J_i^{\text{ref}}\)。当无法同时满足时，则最小化最大的**回报差距** \(g_i(\theta) = \max\{J_i^{\text{ref}} - J_i(\theta), 0\}\)。
- 通过引入KL正则化，将任务采样分布限制在均匀分布 \(p_0\) 的KL球内，得到**极小极大优化目标**：
  \[
  \min_{\theta} \max_{q \in \Delta_k} \left[ \mathbb{E}_{i\sim q}[g_i(\theta)] - \frac{1}{\eta} \text{KL}(q\|p_0) \right]
  \]
  其中 \(\eta\) 是逆温度参数。求解该内层最大化得到最优任务分布 \(q_i^* \propto \exp(\eta g_i(\theta))\)，即 softmax 形式。

### 关键技术细节
- **算法流程（DRATS）**：
  1. 初始化任务分布 \(q\) 为均匀分布，模型参数 \(\theta\)，缓冲区 \(\mathcal{D}\) 为空。
  2. 每轮迭代中，按当前 \(q\) 采样任务并收集轨迹，填满缓冲区。
  3. 使用蒙特卡洛方法估计每个任务的回报 \(\hat{J}_i\)。
  4. 更新参考回报 \(J_i^{\text{ref}}\)（在线估计：初始设为理论最大值，当成功率超过0.5后改为观测到的最大回报）。
  5. 计算归一化回报差距 \(\hat{g}_i = (J_i^{\text{ref}} - \hat{J}_i)/(J_i^{\text{ref}} - J_i^{\text{rand}})\)。
  6. 通过指数梯度上升（mirror ascent）更新任务分布 \(q\)：
     \[
     q_{t+1,i} \propto q_{t,i} \exp\left( \alpha \left[ g_i(\theta) - \frac{1}{\eta} \left( \log\frac{q_{t,i}}{p_{0,i}} + 1 \right) \right] \right)
     \]
  7. 将 \(q_{t+1}\) 投影到最小概率 \(\varepsilon\) 约束的单纯形上。
  8. 使用在线策略算法（如REINFORCE或PPO）更新 \(\theta\)，目标为最大化 \(\mathbb{E}_{i\sim q}[g_i(\theta)] - \frac{1}{\eta}\text{KL}(q\|p_0)\)。
- 算法仅修改数据收集方式，可兼容现有的多任务网络架构（如MOORE、Soft Modularization）。

### 理论保证
- 在标准在线学习假设下（θ-player 满足 C-低遗憾），DRATS 以 \(O(1/\sqrt{T})\) 的速率收敛到最优最差回报差距的 \(\epsilon\) 邻域内（Theorem 4.2）。

## 3. 实验设计：使用了哪些数据集/场景、benchmark、对比方法

### 实验环境
- **Gridworld**：4 个难度递增的网格导航任务，用于控制变量测试正迁移和无迁移场景。
- **MuJoCo6**：包含 Swimmer、Hopper、HalfCheetah、Walker2d、Ant、Humanoid 六个连续控制任务。
- **MetaWorld-MT10**：10 个机器人操作任务的基准套件（Sawyer 臂）。
- **MetaWorld-MT50**：50 个任务的扩展套件。

### 对比方法
- **UNIFORM**：固定均匀采样。
- **LEARNING PROGRESS**（基于 ALP-GMM、RIAC）：优先采样绝对学习进步最大的任务。
- **LEARNING POTENTIAL**（基于 PLR）：优先采样 ℓ1 价值损失最大的任务。
- **HARD FIRST**（基于 SMT）：从当前回报最小的未解决任务中均匀采样，并设定了任务解决/不可解的阈值。
- **EASY FIRST**（仅 Gridworld）：先集中解决最简任务，依次推进。

### 评价指标
- 归一化任务回报 \(\tilde{J}_i = (J_i - J_i^{\min})/(J_i^{\max} - J_i^{\min})\)，聚合平均回报衡量数据效率，worst-task 回报衡量平衡性。

## 8. 不足与局限

- **需要参考回报**：DRATS 需要每个任务的目标回报 \(J_i^{\text{ref}}\)。虽然作者提出在线估计方法（初始设为理论最大值，后改为观测最大值），但若智能体陷入低回报平台，可能低估真实最大值，导致欠采困难任务。该方法假设任务间不根本冲突，当冲突严重时用户需手动指定较低参考回报来折中，增加了使用难度。
- **仅适用于离散任务集**：当前假设任务集是离散且已知的。对于连续任务参数空间，文中建议可使用神经网络输出分布参数（如高斯），但未实现验证，是未来工作。
